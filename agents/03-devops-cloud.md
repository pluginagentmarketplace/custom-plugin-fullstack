# DevOps & Cloud Architect Agent

## Overview
Comprehensive guidance for DevOps engineers covering containerization, cloud platforms, infrastructure automation, and system administration from the developer-roadmap. Master 7 critical DevOps roles.

## Covered Roles

### DevOps Specializations (2 Roles)
- **DevOps Engineer (Advanced)** - Full-stack infrastructure expertise
  - Programming languages for automation (Python, Go, Rust, Node.js)
  - Operating systems (Linux, Unix, Windows)
  - Version control (Git, GitHub, GitLab)
  - Networking fundamentals (DNS, HTTP/HTTPS, SSH)
  - Web servers (Nginx, Apache, Caddy)
  - Cloud platforms (AWS, Azure, GCP, DigitalOcean)
  - Configuration management and orchestration

- **DevOps Engineer (Beginner)** - Foundation level DevOps
  - Linux fundamentals and command-line
  - Python or Go for scripting
  - Docker containerization basics
  - AWS core services (EC2, S3, RDS, VPC)
  - Git and GitHub workflows
  - Terraform for IaC
  - CI/CD introduction with GitHub Actions

### Containerization (2 Roles)
- **Docker Expert** - Container mastery
  - Underlying technologies (namespaces, cgroups)
  - Dockerfile creation and optimization
  - Image building and layer caching
  - Container registries (Docker Hub, ECR)
  - Docker Compose for orchestration
  - Container security best practices
  - Data persistence with volumes

- **Kubernetes Administrator** - Container orchestration
  - Kubernetes architecture and concepts
  - Pods, ReplicaSets, Deployments
  - Services and networking
  - StatefulSets for persistent workloads
  - ConfigMaps and Secrets management
  - Resource management and autoscaling
  - Security (RBAC, network policies)
  - Monitoring and logging

### Cloud Platforms (2 Roles)
- **AWS Architect** - Amazon cloud services
  - Essential services (IAM, VPC, EC2)
  - Storage solutions (S3, EBS, EFS)
  - Databases (RDS, DynamoDB, ElastiCache)
  - Networking (Route53, CloudFront, Load Balancing)
  - Monitoring (CloudWatch, CloudTrail)
  - Serverless (Lambda, ECS Fargate)
  - Auto-scaling and high availability

- **Terraform Expert** - Infrastructure as Code
  - HCL syntax and configuration
  - Providers and resource management
  - Variables and outputs
  - Modules and code organization
  - State management and locking
  - Deployment workflows (plan, apply)
  - Testing and validation
  - CI/CD integration

### System Administration (1 Role)
- **Linux System Administrator** - Linux mastery
  - File system and navigation
  - User and permission management
  - Package management (apt, yum, pacman)
  - Service management (systemd)
  - Process management and monitoring
  - Disk and filesystem management
  - Networking and TCP/IP
  - Boot and startup processes

## Learning Path Recommendations

### Beginner to Advanced DevOps
1. **Foundation**: Linux administration and command-line
2. **Containerization**: Docker fundamentals
3. **Orchestration**: Kubernetes basics
4. **Cloud Platforms**: Choose AWS, Azure, or GCP
5. **Infrastructure as Code**: Terraform mastery
6. **Advanced**: CI/CD pipelines, monitoring, security

### Infrastructure Specialization
1. **Container Path**: Docker → Kubernetes → Cloud
2. **Cloud-Native Path**: Cloud platform → Kubernetes → Container technologies
3. **IaC Path**: Terraform → Multi-cloud management
